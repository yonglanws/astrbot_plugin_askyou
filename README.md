# 问答插件 askyou (astrbot_plugin_askyou)

基于模式匹配的 AI 问答 AstrBot 插件。当消息以「名字 + 触发关键词」为前缀时，自动提取问题内容并调用 LLM 返回回答。

## 功能特性

- **模式匹配触发**：支持 `${名字}${关键词}` 前缀格式自动匹配（如 `akt认为...`）
- **多名字/多关键词**：可配置多个名字和多种判断句式（认为/觉得/说/回答等）
- **模糊匹配**：可选忽略大小写和空格差异（默认开启）
- **LLM 集成**：支持指定 Provider ID 或使用当前会话默认提供商
- **服务降级**：LLM 调用失败时返回友好的降级提示
- **消息截断保护**：超长问题自动截断并提醒用户
- **热更新配置**：无需重启插件即可刷新配置
- **管理指令**：`/ask_stats`、`/ask_reload`、`/ask_test`
- **完整日志**：分级日志记录，便于排查问题
- **调用统计**：实时追踪成功率、延迟等指标

## 使用示例

```
用户: akt认为我今天可不可以赶上公交车
机器人: 你今天可以赶上公交车，建议提前10分钟出发。

用户: 小明觉得明天会下雨吗
机器人: 根据天气预报，明天有70%的概率下雨，记得带伞哦！

用户: AI助手回答1+1等于几
机器人: 1+1 等于 2。
```

## 安装

### 方法一：通过 AstrBot 插件市场安装

在 AstrBot 管理面板 -> 插件市场 -> 搜索 `askyou` -> 安装

### 方法二：手动安装

1. 下载本插件到 AstrBot 的 `data/plugins/` 目录
2. 重启 AstrBot 或在管理面板重载插件
3. 在插件配置中设置名字列表和 LLM 提供商

## 配置说明

在 AstrBot 管理面板的插件配置页面可直接编辑以下选项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `names` | list | `["akt"]` | 触发匹配的名字列表 |
| `trigger_keywords` | list | `["认为"]` | 触发关键词（判断句式） |
| `llm_provider_id` | string | `""` | LLM 服务商 ID（留空用默认） |
| `system_prompt` | text | （见默认） | 发送给 LLM 的系统提示词 |
| `fallback_message` | string | （见默认） | LLM 不可用时的降级回复 |
| `enable_fuzzy_match` | bool | `true` | 是否启用模糊匹配（忽略大小写/空格） |
| `max_question_length` | int | `500` | 问题最大字符数限制 |

## 管理指令

| 指令 | 说明 |
|------|------|
| `/ask_stats` | 查看 LLM 调用统计（总次数/错误率/平均延迟） |
| `/ask_reload` | 热更新配置（无需重启） |
| `/ask_test [问题]` | 测试 LLM 连通性 |

## 架构设计

```
main.py
├── MatchResult          # 匹配结果数据类
├── PatternMatcher       # 消息模式匹配引擎
│   ├── 正则预编译（高性能）
│   ├── 模糊/精确匹配双模式
│   ├── 超长消息自动截断
│   └── 热更新配置（reload_patterns）
├── LLMCallResult        # LLM 调用结果数据类
├── LLMService           # LLM 调用服务
│   ├── Provider 选择策略（指定/默认回退）
│   ├── 完善的错误处理与降级
│   ├── 调用延迟统计
│   └── 统计信息查询（get_stats）
└── MyPlugin             # 主插件类（Star）
    ├── 全局消息监听（on_message）
    ├── 配置热更新（_reload_from_config）
    ├── 管理指令（stats/reload/test）
    └── 卸载清理（terminate）
```

## 单元测试

```bash
pytest test_main.py -v
```

覆盖范围：
- 基础匹配 / 无匹配 / 空消息过滤
- 模糊匹配（大小写、空格）vs 精确匹配
- 多名字 × 多关键词笛卡尔积组合
- 超长消息截断（边界条件）
- 配置热更新（全量/部分更新）
- 特殊字符 / Unicode / Emoji
- 边界情况（前缀包含关系、单字符等）

## 注意事项

1. 名字列表为空时不会匹配任何消息
2. 关键词之间若存在前缀关系（如"认"/"认为"），较短的优先匹配
3. 问题超过 `max_question_length` 会自动截断并提示用户
4. LLM 服务不可用时返回 `fallback_message` 中配置的降级文本
5. 未命中的消息会被放行，不影响其他插件的正常工作
